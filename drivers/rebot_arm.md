# reBot Arm B601-DM Profile

This file describes the Seeed reBot Arm B601-DM 6+1 DoF capability for the Critic to verify proposed actions against.

## Identity

- **Name**: reBot Arm B601-DM
- **Type**: 6+1 DoF robot arm (Damiao motors, LeRobot/ROS2/Isaac Sim compatible)
- **Driver/Profile**: `rebot_arm`
- **Vendor**: Seeed Studio (https://www.seeedstudio.com/reBot-Arm-B601-DM-Bundle.html)

## Degrees of Freedom

6 arm joints — shoulder pan, shoulder lift, elbow flex, wrist pitch, wrist yaw, wrist roll — plus a 1-DoF parallel-jaw gripper. Per-joint limits are enforced by the Damiao motor firmware; this profile lists logical ranges for Critic-side validation only.

## Sensors

- **Joint encoders**: per-Damiao-motor absolute position feedback
- **Current/torque feedback**: Damiao motors expose current sensing usable as a stall-detection heuristic
- **Camera**: optional wrist-mounted (driver-dependent; not included by default)

## Supported Actions

| Action | Parameters | Mock | Real HW | Description |
|---|---|---|---|---|
| `home` | (none) | ✅ | ✅ | Move to default home pose |
| `move_to_joints` | `joints: [j1..j6]` (radians) | ✅ | ✅ | Direct joint-space command |
| `set_gripper` | `value: float ∈ [0.0, 1.0]` | ✅ | ✅ | Position-controlled gripper (0 = open, 1 = closed) |
| `move_to_pose` | `pose: [x, y, z]` (world coords, metres) | ✅ | ❌ | Mock-only Cartesian command (no IK on real hardware) |
| `grasp` | `target_id: str`, `pose: [x, y, z]` (optional) | ✅ | ❌ | Mock-only composite (move + close); also enforces payload limit |
| `release` | (none) | ✅ | ❌ | Mock-only (open gripper + drop tracking) |

On real hardware, the only movement primitives are `home`, `move_to_joints`, and `set_gripper`. Higher-level actions return an error directing the caller to compose them from joint-space commands. This matches the SO-101 driver convention.

## Physical Constraints

- **Reach envelope**: 0.767 m radius from shoulder (per Seeed spec sheet)
- **Repeatability**: 0.2 mm
- **Max payload**: 1.5 kg
- **Shoulder offset (local)**: (0.0, 0.0, 0.40) above base mount point
- **Speed limits**: enforced by Damiao motor controller; software does not override
- **Collision policy**: software-side reach check; no built-in collision avoidance — caller (Critic) must verify against scene
- **Workspace bounds**: not declared at profile level — set per deployment in the scene's `ENVIRONMENT.md`

## Connection

- **Transport**: USB-to-CAN (Damiao 43 USB2CAN Driver Board) → CAN bus to Damiao motors
- **Default port**: `/dev/ttyUSB0` (Linux) or `/dev/tty.usbmodem*` (macOS); override per platform
- **Motor model**: `DM4310` (default) or `DM4340P` (higher-torque variant). Configured via driver `motor_model` parameter.
- **Calibration**: Loaded at connect time from `~/.cache/huggingface/lerobot/calibration/robots/rebot_arm/rebot_arm.json` by default. Generated by Damiao calibration tooling or LeRobot calibrate. *(TODO: hardware verify — confirm canonical calibration tool for the B601-DM bundle.)*
- **Auth**: none
- **Reconnect Policy**: manual on driver init; `close()` disconnects cleanly
- **Optional dep**: real-hardware mode requires the Damiao Python SDK (`pip install damiao_python_sdk`). Mock mode has no extra deps.
- **Backend selection**: `REBOT_ARM_BACKEND=damiao` (default) or `REBOT_ARM_BACKEND=lerobot` (pending hardware verification of LeRobot integration claim).

## Runtime Protocol

- **Arm state channel**: `robots.<robot_id>.arm` in `ENVIRONMENT.md`
  - `joint_angles_rad: [j1..j6]`
  - `gripper_state: "open" | "closed"`
  - `gripper_position: float ∈ [0.0, 1.0]`
  - `holding: <object_id> | null`
  - `end_effector_world: [x, y, z]`
  - `motor_model: "DM4310" | "DM4340P"`
  - `stamp: <iso8601>`

## Authoring Rules

- This profile lists capabilities only. Live state belongs in `ENVIRONMENT.md`.
- The Critic uses the `Supported Actions` table to reject actions with the wrong shape, and the `Physical Constraints` section to reject out-of-envelope poses or overweight grasps.
- `# TODO: hardware verify` markers in the driver source flag values pending validation against the physical bundle. Treat them as conservative defaults until verified.
